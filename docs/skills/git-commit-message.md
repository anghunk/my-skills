# Git Commit Message

自动生成符合 Git 提交规范的提交信息。

## 功能描述

`/git-commit-message` skill 会读取暂存区的变更，并结合项目的最近提交风格，自动生成符合规范的提交信息。

## 使用场景

- 需要保持代码提交记录一致性的团队项目
- 希望提高提交信息质量的个人项目
- 不确定如何描述当前变更内容时

## 使用方法

在 Git 工作目录中，执行以下步骤：

1. 使用 `git add` 将需要提交的文件添加到暂存区
2. 在 Claude Code 中输入 `/git-commit-message`
3. AI 会分析暂存区变更并生成提交信息建议
4. 确认后使用建议的提交信息进行提交

## 提交信息规范

该 skill 遵循常见的 Git 提交信息规范，通常包含以下部分：

- **类型**：feat, fix, docs, style, refactor, test, chore 等
- **范围**：影响的模块或功能
- **描述**：简短描述变更内容
- **详细说明**：更详细的变更说明（可选）

## 示例

```
feat: 添加用户认证功能

- 实现 JWT token 生成和验证
- 添加登录和注册 API 端点
- 更新用户模型以支持密码哈希
```

## 注意事项

- 确保有文件已添加到暂存区
- 提交信息会参考项目的历史提交风格
- 如果不满意生成的提交信息，可以手动修改
